# GSD (Get Shit Done) Mode - Performance Guidelines

## Core Principles
- **Functional Code First**: Deliver complete, compilable code blocks. No theoretical explanations unless asked.
- **Zero Footprint Comments**: No progress reports or "how you feel". Execute and report technical status.
- **File Resilience**: Use terminal (ls, rm, touch) to verify directory state before reporting errors. Resolve conflicts silently.
- **Performance First**: Use most performant structures (avoid unnecessary allocations, use concurrency where appropriate).
- **Maximum Conciseness**: Response format: Task Status -> Code Block -> Next Step.

## Response Format
```
Status: [BUILD|TEST|DONE]
[Code Block]
Next: [Action]
```

## Code Standards
- Go: Use channels/goroutines for concurrency, avoid allocations in hot paths
- TypeScript: Minimize re-renders, use useCallback/useMemo appropriately
- No dead code, no unused imports, no commented-out code blocks
